## ?? What's Wrong with Your Current File

**Your Current admin.license:**
```
AdminID=TENINFOTECH
AdminKey=8C8E4E0A4F5C6B3D9E2A1F7C5B8D4E6A9C3F1E7B5D2A8C6E4F9A1D7B3E5C8F2A
ValidUntil=2030-12-31
```

**The Problem:**
The AdminKey doesn't match the SHA256 hash of `"TENINFOTECH|DIGISIGN_ADMIN_SECRET"`

**Correct admin.license will be generated by the script!**

## ?? Alternative: Manual Steps

If the batch script doesn't work, try these manual steps:

### Option A: Using Command Prompt
```batch
# Compile the tester
csc /out:AdminKeyTester.exe AdminKeyTester.cs

# Run it
AdminKeyTester.exe

# Copy generated file
copy admin.license D:\Development\DigiSign\
```

### Option B: Using PowerShell
```powershell
# Generate the correct AdminKey
$adminId = "TENINFOTECH"
$data = "$adminId|DIGISIGN_ADMIN_SECRET"
$sha = [System.Security.Cryptography.SHA256]::Create()
$hash = $sha.ComputeHash([System.Text.Encoding]::UTF8.GetBytes($data))
$key = [BitConverter]::ToString($hash).Replace("-", "")

# Create the file
@"
AdminID=$adminId
AdminKey=$key
ValidUntil=2030-12-31
"@ | Out-File -FilePath "D:\Development\DigiSign\admin.license" -Encoding ASCII

Write-Host "? admin.license created!" -ForegroundColor Green
```

## ?? Troubleshooting

### Error: "csc is not recognized"
**Solution:** Add .NET Framework to PATH or use Visual Studio Developer Command Prompt

```batch
# Find csc.exe location
dir /s C:\Windows\Microsoft.NET\Framework\csc.exe

# Example paths:
C:\Windows\Microsoft.NET\Framework\v4.0.30319\csc.exe
C:\Windows\Microsoft.NET\Framework64\v4.0.30319\csc.exe
```

### Error: Still getting "Invalid admin license"
**Check these:**
1. File name is exactly `admin.license` (no .txt extension)
2. File is in same folder as DigiSign.exe
3. File encoding is ASCII or UTF-8 (not UTF-16)
4. No extra spaces or hidden characters
5. Date format is exactly YYYY-MM-DD

### How to verify file encoding
```batch
# Open in Notepad
notepad D:\Development\DigiSign\admin.license

# Save As ? Encoding: ANSI or UTF-8
```
